Сейчас протокол немного "подправлен для красоты". Чтобы данные о весе, номере контейнера и пломбах
выглядели единообразно во всех случаях регистр качества и статуса предшествует данным.

К протоколу
(везде далее порядок байт little endian, то есть до пнрекодировки в modbus или после декодировки)

Номер байта
-------------веса------------------
0			норма/недовес/перевес
1			ошибка
2			стабильность
3			здесь всегда 0
4			вес брутто 0
5			вес брутто 1
6			вес нетто 0
7			вес нетто 1
8			вес тары 0
9			вес тары 1
-----------контейнер----------------
10			авто/ручной
11			ошибка
12			число рег. номера 0 \
13			число рег. номера 1 / число=n
14--13+2*n  сам номер в виде строки без 0 на конце
	(=k)	(с нулем, если занимае нечетное число байт)
-------------пломбы----------------
k+1			число пломб 0
k+2			число пломб 1
	----------- пломба_1----------
k+3			номер пломбы 0
k+4			номер пломбы 1
k+5			авто/ручная
k+6			ошибка
k+7			число рег. пломбы 0 \
k+8			число рег. пломбы 1 / число=n
k+9--(k+8)+2*n	сама пломба в виде строки без 0
	---------конец пломба_1--------
