Данный вариант драйвера может работать с терминалом DIS2116 через com2 терминала. О постоянно ведет опрос выдавая команду MSV и транслирует из Matrix в терминал 2 команды: CDL (аналог set zero) и TAR (установить текущий измеренный вес как вес тары). Для корректной работы единицы должны быть килограммы, десятичной точки в пережаваемых данных быть не должно (это заводские установки, но при необходимости конфигурируется руками в меню терминала).

Записи в конфигурационном файле драйвера
----------------------------------------
Чтобы работать с терминалом нужно

1. В serial/device указать model="dis2". Вот фрагмент
serial {
  dev = "/dev/ttyS1"
  baud = "115200"
  data = "8"
  stop = "1"
  flow = "off"
  parity = "none"

  id = 1
  device {
    addr = 0
    model = "dis2"
  }
}

2. Драйвер может отдавать в Matrix вес нетто или брутто и давать или не давать вес тары. Это конфигурируется полями Net и Gross в секции Gross в соответствии с таблицей
Net			Gross	В сообщении
"0"			"0"		Вес брутто, Тара=0
"0"			"1"		Вес брутто, Тара присутствует
"1"			"0"		Вес нетто, Тара=0
"0"			"1"		Вес нетто, Тара присутствует
Отсутствие поля в разделе трактуется как "0". Так как поля Position и Length не используются, весь раздел Gross можно опустить. Это равносильно конфигурации первой строки таблицы (все в точности как для ldu). Чтобы в Matrix понять, что вес нетто, введено дополнительное битовое поле в status (см файл devdrvd_communication.h). Пример
Gross {
	Position = 7
	Length = 5
	Net = "0"
	Gross = "0"
}
в сообщении будет передаваться вес брутто, поле тары занулено.

3. для обеспечения передачи команд из Matrix в секцию command добавляется раздел модели dis2
command {
  cmd ldu681 {
	set_zero = "SZ\r"
	set_tare = "ST\r"
	reset_tare = "RT\r"
  }
  cmd dis2 {
	set_zero = "Z"
	set_tare = "T"
  }
}
Как уже сказано "Z" инициирует на терминал команду CDL, "T" инициирует на терминал команду "TAR".
