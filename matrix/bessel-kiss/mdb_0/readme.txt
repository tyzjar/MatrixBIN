Запуск

mdb -s 13

Параметр: -s = номер slave сечас не используется (откликается на любой номер, который будет в запросе), но по умолчанию стоит 97.

Сейчас должен отвечать на команды modbus 0x04 и 0x10. Ошибки по modbus пока не реализованы. Запись в регисры есть, но пока не надо трогать, т.к. взаимодействие по mqtt еще нет.

Один вес выдает по такой команде:

0x01,0x02,0x00,0x00,0x00,0x06,0x01,0x04,0x00,0x00,0x00,0x01
Номер slave---------------------^ может быть любой

Если спросить больше регистров, пришлет ответ, но все веса кроме первого будут нулевые.

Внутренняя база данных начинается с регистра 0 (в запросе modbus) и т.д. Здесь храняться веса (сейчас 1 вес - /matrix/wgt)
Регистры для записи начинаются с номера 1024=0x400 (запрос modbus 04 00). Сейчас их 4 штуки. В mqtt публикуются /modbus/set_register/(ri,val).

Поддерживается одно соединение. Если приходит запрос на новое соединение, то считается, что это клиент потерял старое. Поэтому прежнее соединение закрывается, а новое создается и считается актуальным.

Чтобы компилировать нужно поместить директорию mdb на одном уровне с src в simplex-scaled.
